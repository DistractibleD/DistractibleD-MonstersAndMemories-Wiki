Items with possibly missing information
=========================================

This file tracks items where the *data* in items.json might be incomplete — not whether
the saved screenshot's border/crop is complete. Since 2026-07-04 the site builds its own
card from items.json and no longer displays the screenshot at all, so a screenshot that's
visually cut off (missing border, text running to the raw edge) no longer matters on its
own — only whether the actual stats/text made it into items.json. Screenshots are still
kept (as .jpg) purely as the reference used to extract that data, per the user's
2026-07-04 confirmation.

Update (2026-07-04, re-check of every item previously flagged in this file):

- Torch — WAS missing data, now fixed. The card (images/items/torch.jpg) has an "Effect:"
  line ("Torch (Passive. Must Equip. Level: 1)") and a description ("A simple torch.")
  that had never actually been added to items.json (previous notes here were only about
  the image's bottom border being cropped, not about the data itself) — both are now
  added. Two more things on this card don't fit any existing schema field and weren't
  added: "Fuel: 100%" (looks like a consumable/instance resource level, not a fixed item
  property — no other item has shown anything like it) and "Right-click to activate."
  (a green action-flag line, similar to a container's "Right-click to open." but not
  currently tracked as a field for any item type). Flagging these in case a future card
  clarifies whether they're worth a schema field.
- Stained Black Iron Dagger — data is complete (full 14-class list, race, description all
  present in items.json). The earlier note here was only about the saved screenshot being
  visually cut off, which no longer matters since the screenshot isn't displayed. No action
  needed.
- The Measure Stick — data is complete (effect, description, weight/class/race all present,
  merged from a two-screenshot capture back when screenshots were still the display). No
  action needed.
- Mining Pouch — data is complete; the earlier note was only about the image's cropped
  border. No action needed.

Nothing else has been flagged as incomplete. This file will be updated only when there's
a specific reason to believe an item's data (not its screenshot) is missing something.
